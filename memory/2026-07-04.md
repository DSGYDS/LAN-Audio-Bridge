# 2026-07-04 每日笔记

## MicSwitcher 废弃，改用 B 方案

### 背景
之前尝试用 COM IPolicyConfig 切换系统默认麦克风到 CABLE Output，
但该接口是未公开的 Windows 内部 API，在不同 Windows 版本上不稳定，
且已在开发过程中触发进程级 AccessViolation。

### 决策
**放弃 A 方案（COM IPolicyConfig 切换默认麦克风），
采用 B 方案（直接用 WasapiOut 写 CABLE Input）。**

### 原理
VB-CABLE 虚拟音频设备内部是环回缓冲区：
- CABLE Input（Render 端点）← 应用写入音频
- CABLE Output（Capture 端点）← 驱动自动复制 Input 数据

所以不需要切换默认麦克风，直接把解码后的音频写进 CABLE Input 即可。

### 影响
- 用户需在目标应用（微信/OBS/游戏）中手动选择 CABLE Output 作为麦克风
- 一次配置，永久有效
- 不影响系统其他应用的默认麦克风
