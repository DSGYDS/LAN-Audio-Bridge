### 核心架构变化

**重构前：**
```
AudioEngine ←── UdpClient(12345)
HandshakeServer ←── UdpClient(12347)
AudioPipeline ←── UdpSender(→12345)
```

**重构后：**
```
AudioEngine ←── ITransport (UdpTransport, 12345)
HandshakeServer ←── ITransport (UdpTransport, 12347)
AudioPipeline ←── ITransport (UdpTransport, →12345)
                              ↑
                         PlatformFactory.CreateTransport(Udp, ...)
```

### 关键设计决策

| 决策 | 内容 |
|------|------|
| PacketHeader 不改 | 通过 Adapter 实现 IPacketProtocol，不改原有编解码逻辑 |
| AudioMixer/AudioEncoder 不入接口 | 纯逻辑工具类，不调平台 API |
| 每个端口独立 Transport 实例 | 握手(12347)和音频(12345)各自 UdpTransport |
| IAudioCapturer 纯 PCM 层 | readFrame 返回 PCM16LE，Opus 编解码由 Pipeline 处理 |
| AudioRouter 降级为路由决策器 | 持有 SpeakerRenderer + CableRenderer 两个 IAudioRenderer 实例 |
| Config 逐步迁移 | P2 新建 Config 容器，P4-P6 逐步替代现有 AudioConfig |
| 无 EventBus | 当前无多订阅者需求，先保持简单回调 |
| 跨语言接口各放各的 | C# 和 Kotlin 各自一套同名接口文件，第一阶段不引入共享构建 |
