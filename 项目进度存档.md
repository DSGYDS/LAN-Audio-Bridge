## 项目定位

一句话：手机音频零配置、低延迟、多模式传到电脑播放的局域网工具。

当前成熟度：**核心链路稳定，功能闭环**
- 核心链路已通（Android 采集 → Opus 编码 → UDP → Windows 解码 → 播放）✅

## 🔧 构建与运行

### Windows端
```
cd windows/LanAudioBridge.Desktop
dotnet build -c Release
dotnet run
```

### Android端
```
cd android
./gradlew assembleDebug
```

### 运行环境
- Windows：.NET 10 运行时、VB-CABLE 虚拟音频驱动（archive/ 中含安装包）
- Android：Android SDK、支持 MediaProjection 的设备
- 网络：局域网互通，UDP 12345（音频）/ 12347（控制）端口开放
- 启动顺序：先启 Windows 接收端 → Android 端 mDNS 发现并连接 → 选择模式推流

### 已知限制
- 系统音频采集强制锁媒体音量为0（暴力方案）
- 无真正系统托盘（关窗口=隐藏，需任务管理器退出）
- mDNS 在纯 WiFi Direct（无路由器）环境下可能不可用（重连用 lastKnownHost 兜底）

## 性能参数

| 项目 | 当前值 |
|------|--------|
| AudioRecord缓冲 | ~80ms（FRAME_BYTES * 4） |
| MediaProjection缓冲 | ~640ms（FRAME_BYTES * 32） |
| Win BufferMs | **100ms**（从300逐档下调，稳定最低值） |
| Win DesiredLatency | **100ms**（WaveOutEvent） |
| Win Mic latency | **50ms**（WasapiOut CABLE Input） |
| JitterBuffer | **60ms**（3帧预缓冲，5帧容量） |
| 热切防抖 | **150ms**（debounce，只执行最后一次） |
| 淡入 | 50ms × 48000 = 2400采样点 |
| 音频帧 | 20ms（960采样点 @ 48kHz） |
| 看门狗超时 | 3000ms（触发 Reconnecting） |
| **端到端延迟** | **~200-260ms**（含 JitterBuffer 60ms） |
