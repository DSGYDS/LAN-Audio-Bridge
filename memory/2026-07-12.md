## 延迟调优（18:10~18:45）

### 实施记录
- **第一步**：加入 BufferStats 监控（`OnFrameProcessed`，约 50 行），编译 0 错
- **第二步**：BufferMs 逐档下调 300→150→100，每档听测稳定
- **第三步**：WaveOutEvent DesiredLatency 80→50（卡顿，回退），→100（稳定，保留）
- **第四步**：WasapiOut CABLE Input 80→50（稳定，保留）
- Android buffer 不动（确认不影响稳态延迟）

### 最终值
| 参数 | 最终值 |
|------|--------|
| BufferMs | **100** |
| WaveOutEvent DesiredLatency | **100** |
| WasapiOut DesiredLatency | **50** |

### 经验教训
- DesiredLatency 50 在部分机器上会导致 WaveOutEvent 卡顿（提交粒度太小，Windows mixer 不稳），100 是安全的
- WasapiOut 50 反而稳定，说明 CABLE 驱动的缓冲区处理比 WaveOutEvent 更抗小延迟
- Android buffer size 不等于固定延迟，之前画像表有误，下次做调优画像要注意区分"环形缓冲区"和"排队缓冲"
