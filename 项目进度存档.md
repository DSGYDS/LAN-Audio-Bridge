## 项目定位

一句话：手机音频零配置、低延迟、多模式传到电脑播放的局域网工具。

当前成熟度：**PoC 级别，功能可跑但不够稳**
- 核心链路已通（Android 采集 → Opus 编码 → UDP → Windows 解码 → 播放）✅
- 但存在安全机制缺失、状态机失效、弱网不闭环、工程化不足等多项问题

## 🔍 外部审查结论（2026-07-13）

### 严重问题
1. **无安全机制** — UDP端口接受任意来源包。但局域网场景优先级可下调
2. **状态机失效** — isValidTransition禁止IDLE→CONNECTING但代码直接调，UI显示不准
3. **弱网没闭环** — FEC开了但decodeFec=false；序号传了但不检查；发送失败不处理

### 架构问题（已知卡点）
4. **P4抽象层没接入主链路** — 两套网络实现并存
5. **Android后台保活不完整** — 管线在Activity而非Service
6. **混音无同步机制** — 顺序读取直接相加
7. **配置集中不彻底** — 常量仍多处硬编码

### 工程化问题
8. 强制静音轮询粗暴；无真正系统托盘
9. 无自动化测试/CI
10. **Gradle写死本地路径，他人无法构建** ⚠️
11. 文档与实际不一致（回滚后未同步）
12. VB-CABLE驱动再分发无许可证说明

## 🔧 构建与运行

### Windows端
```
cd windows/LanAudioBridge.Desktop
dotnet build -c Release
```

### Android端（首次需修改Gradle路径）
```
cd android
修改 gradle/wrapper/gradle-wrapper.properties 中distributionUrl为远程地址
./gradlew assembleDebug
```

### 已知问题
- Gradle Wrapper写死本地路径，他人无法构建
- proguard-rules.pro不存在但被build.gradle.kts引用
- 系统音频采集强制锁媒体音量为0（暴力方案）
- 无真正系统托盘，关窗口只能任务管理器杀进程
